I don't think it was what Raz said, because if the AVS window was outside the screen, you wouldn't even see it for a flash. It only becomes visible after the move.

An other option was that there was a bad/corrupt preset active, which would crash AVS as soon as it started. The last preset is saved as vis_avs.dat along with the plug-in (vis_avs.dll), so next time you experience something like that, try deleting vis_avs.dat (not dll).
