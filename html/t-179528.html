I sped it a bit up (and made it look better i think :)) by using a static convolution filter bump instead of the dynamic ones.

if you dont to understand it, here's an explanation:
if you look at the convolution filter kernel (that's the 7*7 table with the numbers in it), you'll see this in center:
-1 -2 0
-2 4 2
0 2 1

this means that if we are for example at pixel(10, 10), the new value for that pixel is the sum of the following lines:
-1 *pixel(9,9)
-2 *pixel(10,9)
0 *pixel(11,9) <--- this line is left out by the convo filter because its factor is zero

-2 *pixel(9,10)
4 *pixel(10,10)
2 *pixel(11,10)

0 *pixel(9,11) <-- this line, too
2 *pixel(10,11)
1 *pixel(11,11)

that new value is then divided by the value you enter for "Scaling".
I hope that was a good explanation :)

now to the effect of that calculation:
if there is a bright bar, it looks casts a shadow to the lower right and has a highlight on the upper left.
