Originally posted by Nanakiwurkz
suggestion try using el-vis 3d engine scope from el-vis real 3d pack the preset is el-vis_real_3d_start
i'd suggest not using that, but instead using the 3d stuff from PAK-9's AVS Programming Guide (google it)

reason: el-vis's real3d pack is nice, but it is old and kind of unoptimized :)
code from "elvis_real3d_beatspiral"

x=x4/(1+z4/dst);
y=y4/(1+z4/dst);

as you can see there are lots of divisions in there, which can really slow down things if used too often

optimisation #1:
we divide by the same divisor twice, so we might as well cache 1/divisor and multiply it with both

optimisation #2:
since dst is only changed once per frame, one can move the divison for that into the frame code box in the much same manner as optimisation #1

frame:
invdst=1/dst;

point:
invz=1/(1+z4*invdst)
x=x4*invz;
y=y4*invz;

there, 3 divisions per point saved

not meaning to criticise you el-vis, i know that's old code, my code from that time looks even worse :)
