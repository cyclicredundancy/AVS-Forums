introducing PimpBot Runtime
new version is available now!

this new version introduces the PimpBot Runtime. as pimpbot creates .pimp files including all settings for the installers and includes all installable data, i thought it could become a good alternative to actual installers.

so rather than compiling an installer, you can take the .pimp file which pimpbot creates anyway and distribute as if it was a zip with your data. just that you get all the comfort from a pimpbot installer. this means smaller downloads and no more suspicion against executables. the downside of course is that the user requires pimpbot runtime.

at this time it only ships with pimpbot, later it will be available as a seperate download. the main reason for this is, that it doesn't install apes or fonts yet. the apes will btw be installed with pimpbot runtime, in order to keep the .pimp files small.

by default, .pimp files still get opened in pimpbot for modification/recompilation. to install the contents, you have to do a right-click and press on 'install'.

bugs and suggestions can be posted here. enjoy!

PimpBot Runtime beta
pimpbotRT went beta and can be downloaded here (includes examples).

what is it again?
all new pimpbot version store the data of your installer in a .pimp file, so you can recompile/edit it easily at any time. so why not distribute the .pimp files directly? this is where pimpbotRT comes in. loading a .pimp file into pimpbotRT will create an installer on runtime, hence the name. the benefit? smaller files, less false virus reports.. or you can just unzip the files with any 7-zip compatible tool (i.e. winrar).

pimpbot 3.2 beta
it's been a while and there might not be much reason left to update pimpbot, but i did so anyway. let me sum up the news.

1. DROPFOLDER
though pimpbot already makes it very easy to create an installer for your presets, i tried to make it even easier. stick to certain conventions and you can simply drop your project-folder on the pimpbot icon and it will detect the contents automatically. check attachment for details!

2. BACKUPS
pimpbot now includes a backup tool. at the moment you only get a stripped down version of syncbot. it allows you to make offline and online backups of your presets and your previous pimpbot compilations. will get better features and will be integrated in the compiler.

3. TWEAKS
so far, pimpbot's default settings suited my personally preferred settings. you can easily change this by launching pimpbot with the /defaults parameter. this will disable all the checks pimpbot usually does and allows you to put your prefered defaults in any of the fields pimpbot offers. next time you launch an empty pimpbot project, your defaults will be used.

go ahead and download pimpbot!

did you install winamp properly or did you just copy your old installation? did you save that program files path in your registry or is it a directory you just created and have to select manually for each installation?

pimpbot installers use the registry keys created by the winamp installer to determine the correct path. if that fails, it assumes %PROGRAMFILES%\Winamp to be the path.

you can always use the /unzip switch to extract the presets from the installer (Installer.exe /unzip) or the nsis default /D to override the destination (Installer.exe /D=F:\Program Files\Winamp).
