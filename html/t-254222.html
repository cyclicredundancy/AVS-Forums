Based on that information, I'd be more inclined to believe the problem is with JTFE and not AVS... But still, I find it incredibly odd that AVS would be the only visualization plugin affected by this!

The JTFE installer didn't accidentally include one of those obsolete APEs that's been known to cause problems, did it? Or, worse, mess with the AVS plugin DLL itself?

This would be easier if we had some sort of error message.

Here, try a little experiment: Look in your Winamp\Plugins folder for a "vis_avs.dat" file. This is a copy of the current preset, and if you delete it, AVS will have a blank preset next time you start it up. If you still get a crash when you start up AVS, then the problem is... well... it's something else. You may have to wait until JTFE build 50 or something.
