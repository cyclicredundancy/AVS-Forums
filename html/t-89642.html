Well it's probably the same in Milkdrop and AVS (though they might be implemented a bit differently, I don't know), but the main problem is that components like the Superscope and Dynamic movement evaluate your code as you type it. So when you type:

red=blue

AVS will register the variables r, re, red and b, bl, blu, blue.

The library doesn't remove unused variables (or AVS doesn't), so the only way to get rid of them is to save and reload your preset. I've also noticed that sometimes (when you already have a lot of variables) AVS won't register new ones. Again, the only fix is save/reload.

Milkdrop doesn't have this problem because you have to hit CTRL-ENTER to apply your code (or at least it doesn't suffer from it as much). I don't think there's ever been a MD preset that hit the limit. In AVS though it's much easier, for example if you wish to store a point in a 3D superscope, you have to use 3 variables (x,y,z). It adds up easily.

I guess you could try and see how many variables MD and AVS can take... you'll know you hit the limit when it crashes :D
