It started doing this after I had upgraded to the latest version of winamp. When I looked at the windows error report it said that colormap.ape was the problem. StevenRoy
30th August 2005 05:15 GMT

You can rename the "colormap.ape" file, which will effectively disable the "Color Map" effect in all your presets. Then you should be able to remove it from your preset (in case some of the data was corrupted somehow).

It's also possible, though, that the problem is in your "colormap.ape" file instead of your preset. You should confirm that other presets that use "Color Map" still work correctly.

Try getting a preset pack that has colormap in the installer (most installers that were made with pimpbot have it).
The installer should install a new one if colormap isn't in the /avs dir. (which you have moved to another directory)

If your old colormap was corrupted, this should fix it.

If the problem persists, the problem lies somewhere else. (obvious, but true :p)
