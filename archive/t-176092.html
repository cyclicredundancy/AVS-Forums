Originally posted by sandstorm_sh
i just wanna rewrite it into java??
*grins*
I think AVS is a bit too complex to be rewritten.

The hardest thing will probably be convincing all APE authors to rewrite their plugins in Java :)

Then you need a code parser, which is really hard, but maybe you can find one already written in Java. After all, the AVS syntax is not too far away from Java/PHP/Perl/C. But then, to make the whole thing effective, you'll *need* to compile the script parts to bytecode, so they can be executed on demand
This alone needs in-depth knowledge of the Java VM.

Next, you'll need to know how all the standard effect components of AVS work (like blur, dynamic movement, etc.)

If you're then done with all that work (after maybe 5 years or so :) ), Nullsoft will probably have released several new versions of AVS, and your work goes on :)
