A short constructive review:
First One - Why have such a nice scope if you cover it up with that movement? at least set that movement filter to blend.
This is like installing a Lada engine in a Porsche :igor:

Inside the cockpit
generally:
Too fast my eyes!!!
Technically:
The color reduction:
I don't see the effect of it.

The ssc: having n set to 1100 is a bit overdone in my eyes. 400 or 500 should be plenty IMO.
tpi.. $pi is integrated now so you don't need to use acos(-1) anymore.
remove Rot and Point. Rot is constant, so there's no need to set them to 0 each frame, especially if they aren't even used:
calculating x1 and y1 the secondary time like this is useless, especially if you don't include point=point+1; per point.

DM: Rot is defined but not used. better remove it.
x and y are static, so there's no need for a >dynamic< movement. Just use a normal movement with the same settings.
a small detail: multiplying is faster than dividing. use *0.5 instead of /2

Also, you entire preset is inside a ignore/replace effectlist. move everything outside it because effectlists are slow. Creating AVS presets technicalwise is all about efficiency.

My advice: Read the tutorials, because it seems you copypasted half your code here to me.
