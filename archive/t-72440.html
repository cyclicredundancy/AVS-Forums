When you set a beat customizer to reverse, everything after that object that uses the beat will reversely-detect the beat.

That is, you have a scope that changes from blue to red on beat, but then place the reversal before it, it will now change from red to blue every beat. However, if you have a function in the onbeat box of the scope, that function will be done every time (every frame) EXCEPT when there's a beat.

I don't find that it's useful for most effects, because they are usually set to do what you want them to do on beat, rather than set them to do something when there's NOT a beat. I suppose there could be a use if you have a render/trans object between two customizers (reversals) so that the following parts of the preset are not effected, but nothing comes to mind...
