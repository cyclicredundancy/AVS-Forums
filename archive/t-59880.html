I feel your pain rradical!

I've found a way to get around it. If you run two video cards, you can run AVS windowed on the second card and select the window at the edge of your screen. Then the Hotkeys will function. I'm pretty sure that the hotkeys not functionning is somehow related to the problem I describe below. If the Avs window would stay "open" as a tab at the bottom of the screen of it's own as it does when winamp minimizes and Overlay mode isn't on, this would make the problem easier to avoid.

I sort of got around this by using a two-monitor setup where one monitor is my main desktop and the second is a secondary desktop which is where I run the AVS window fullscreen to display through the projector. This way you pick where you want the second desktop to be(top, bottom, right, left) and just run the mouse off the main scren to get to it. If you bring the mouse just to the edge of the screen you can "select" the displaying window without the mouse showing up. All you need is a motherboard that has AGP and PCI slots, I'm running a 3dProphet II mx as the main card and an Ati mach 64 as the secondary with the TV-out.


Have you also noticed that there is a small glitch? If you're running AVS in a window and running it in overlay mode, the AVS window will minimise with Winamp regardless of the "do no minimize on winamp minimise" option being checked. Perhaps it's not a glitch, perhaps it's necessary, but it really would be great if you could still select whether it minimized or not with the overlay running.
