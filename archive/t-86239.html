AVS Idea - Screen Choices
Dear Winamp3rs,
I have been using Winamp3's Advanced Visualisation Studion with my graphics card's TV-Out facility, but if I click Go Fullscreen it appears only on my monitor. I had to move the visualisation window onto the tv (I used extend my desktop onto this monitor) which meant I had the skin border around it which was annoying.

I think that there should be a facility to select which screen the "Go Fullscreen" button sends the image to, so that people who either have TV-Out or Dual Head graphics cards can choose which to show the visualisation on, e.g. if you have a party and set up a projector on the second output pointing at the wall and have the first output connected to a monitor with the WinAmp control window on it so you can control playing/browse media library e.t.c. without having to dismiss the fullscreen visualisation.

Please do not hesitate to express your thoughts on this matter.

CooCoo signing off.

Multiple Screens and fullscreen
I have just setup my new beast for party graphics intending to get the same functionality that you are looking for. AMD1700/Geforce4/XP Pro

I have found that it is best to run fullscreen on display#1 and the controls on display#2 otherwise winamp/xp freaks out big time.

So far its not been robust enough for use in a party where you need a crash free professional look. What I have found is that I can get a nice clean fullscreen running on my primary display and the controls on the secondary. I have loaded my fav presets to hotkeys and can mix it up quite nicely as long as I am not running fullscreen. Because as soon as you press a key to load a preset or open another application on the other screen you drop back to normal window display. VERY ANNOYING

One solution that I plan to investigate is to try and find a program that lets you select an area of a desktop to be output to the TV OUT. That way you can run the visuals in a small screen and mix it up. It also provides much better performance than fullscreen.

Does such a program exist? Where to start?

The other solution would be for Nullsoft to provide more functionality/support for dual head video cards in a forthcoming release.
