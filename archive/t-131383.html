Firstly, please get back to us on whether AVS is still doing its thing in the background (check the CPU usage as mentioned above).

If AVS is still running, then the window is probably outside your screen or something. You could ask some of the wasabii gurus on the winamp3 forum where this info is stored and how you can set it back to default.

If AVS isn't running anymore, look for the file "vis_avs.dat" in your Winamp3\wacs directory. This file contains the last preset you where working on: it might be corrupt, causing AVS to crash/shutdown immediately. Try deleting it; this file only contains a copy of the preset, so if you saved it as a separate filename it won't be lost (it doesn't hurt to try this).
